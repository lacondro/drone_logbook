# WIZGENE Drone Logbook 사용 설명서

PX4(`.ulg`) / ArduPilot(`.bin`) 비행 로그를 모아 **정리·검색·기록**하는 사내 웹앱입니다.
폴더를 가리키면 모든 로그를 자동으로 읽어 리스트로 보여주고, 로그별 기체·비행 요약과
GPS 지도를 확인하며, **조종사·기체·특이사항**을 기록할 수 있습니다.

> 이 도구는 **로그북(기록부)** 이지 분석 툴이 아닙니다. 시계열 플롯·PID·FFT 같은 상세
> 분석은 [PX4 Flight Review](https://review.px4.io/) /
> [ArduPilot UAV Log Viewer](https://plot.ardupilot.org/) 를 쓰세요.

---

## 1. 접속과 화면 구성

- 사내 NAS에 배포된 경우: 브라우저에서 **`http://<NAS-IP>:8137`** 접속.
- 상단 메뉴 3개:
  - **Flights** — 비행 로그 목록/상세 (메인 화면)
  - **Aircrafts** — 기체 관리 (등록번호·상태 등)
  - **Pilots** — 조종사 명단/집계

---

## 2. 먼저 알아둘 핵심 개념

### 로그북 폴더와 저장 위치
- 앱은 **서버(NAS)의 로그 폴더**를 읽습니다. NAS 배포 시 기본 폴더는 **`/logs`** 로
  고정되어 있고, 들어가면 자동으로 그 폴더가 잡힙니다.
- 조종사·특이사항·기체 등록정보 같은 **사람이 입력한 데이터와 파싱 캐시**는 그 폴더 안의
  **`flightlogbook.db`** 파일에 저장됩니다. 즉 **로그와 기록이 같은 곳에** 보관됩니다.

### 컨트롤러(hwid) ≠ 기체(에어프레임) — 중요
- 로그에서 얻는 식별자는 **비행 컨트롤러(픽스호크)** 의 것(`hwid`)입니다. 에어프레임이
  아닙니다.
- 그래서 **같은 컨트롤러를 여러 기체에 옮겨 달면** 로그상 식별자는 같습니다. 앱은 이를
  분리해서, **기체(에어프레임)는 사용자가 지정**하도록 되어 있습니다. (→ 5.6 참고)

---

## 3. Flights 탭 — 로그 불러오기

화면 맨 위에 **필터 줄**과 **스캔/업로드 줄**이 한 줄(5:5)로 있습니다.

### 3.1 Scan (서버 폴더 읽기)
1. 오른쪽 입력칸에 **서버에 있는 폴더 경로**를 넣습니다. (NAS는 보통 `/logs`가 자동 입력됨)
2. **Recursive**(하위 폴더까지) 체크 — 보통 켜둡니다. 하위 폴더(`/logs/px4/...`,
   `/logs/2026-06/...`)도 전부 훑습니다.
3. **Scan** 클릭.

> 경로는 **서버(NAS) 기준**입니다. 브라우저(내 PC)의 `C:\...` 경로를 NAS 앱에 입력해도
> 안 됩니다 — NAS는 내 PC 디스크를 못 읽습니다. 내 PC 파일을 올리려면 **Upload**를
> 쓰세요.

스캔 결과 메시지 예: `Scanned 120 · new 8 · cached 112 · failed 0 · dup 2`
- **new**: 새로 읽은 로그, **cached**: 안 바뀌어 건너뛴 로그, **failed**: 파싱 실패,
  **dup**: 중복으로 건너뜀, **removed N missing**: 사라진 파일의 항목 자동 정리.

### 3.2 Upload (내 PC 로그를 서버로 올리기)
- **Upload** 버튼 → 내 PC의 `.ulg`/`.bin` 파일을 **여러 개 선택** → 서버 `/logs`에
  저장되고 **자동으로 스캔**됩니다.
- NAS처럼 서버가 내 PC를 못 읽는 환경에서, **원격으로 로그를 올릴 때** 유용합니다.
- 로그가 아닌 파일은 자동으로 무시됩니다.

## 4. Flights 탭 — 찾기 (필터·검색·정렬)

### 4.1 필터
- **All vehicles**: 특정 기체만 보기 (등록번호/별명 기준).
- **All stacks**: PX4 / ArduPilot 만 보기.
- **연도-월-일 ~ 연도-월-일**: 날짜 범위.
- **Search** 칸: 파일명 · 조종사 · 특이사항 · **위치(시군구, 예: 강동구)** ·
  **컨트롤러 hwid** · 등록번호로 검색.
- 필터가 걸리면 **Clear** 버튼으로 한 번에 해제.

### 4.2 정렬
- 리스트 헤더 오른쪽에서 **Date / Duration / Distance / Pilot / File** 기준 정렬,
  **▲/▼** 버튼으로 오름/내림차순.

### 4.3 상태 배지(목록의 색 표시)
- **PX4 / ArduPilot**: 비행 스택.
- **ok**(초록): 정상, **warn**(노랑): 경고 메시지 있음, **error**(빨강): 에러/파싱 실패,
  **partial**: GPS 없어 위치·시간·거리 일부 없음.

---

## 5. Flights 탭 — 상세 보기

목록에서 카드를 클릭하면 오른쪽에 상세가 열립니다(자체 스크롤). 4개 블록:

### 5.1 Vehicle & Firmware
기체/펌웨어 정보 — Airframe, Hardware, Software/OS Version, Estimator, Logging
Start/Duration, Start Location(시군구), **Flight Controller (HW ID)** 등.

### 5.2 Flight Statistics
Distance, Max Altitude Difference, Average/Max Speed, Max Tilt 등 비행 요약.

### 5.3 Flight Track (지도)
- Leaflet 위성지도에 **GPS 트랙**(초록=시작, 빨강=끝) 표시.
- 우측 상단 **Map view / Satellite** 버튼으로 지도 모드 전환.
- GPS가 없는 로그는 "No position data"로 표시됩니다.

### 5.4 Logbook Entry (직접 입력)
- **Pilot**: 조종사 이름.
- **Assigned vehicle**: 이 비행의 **기체(에어프레임)** 지정/변경. 같은 컨트롤러라도
  다른 기체로 바꿀 수 있습니다.
- **Remarks**: 특이사항 메모.
- **Save entry** 로 저장. (이 정보들은 로그 파일엔 없고 DB에 저장됨)
- **Delete flight**(빨강): 이 비행을 로그북에서 제거 + **서버 디스크의 로그 파일까지
  영구 삭제**. 되돌릴 수 없으니 확인창에서 한 번 더 확인합니다.

> 단순히 목록에서만 지우면 다음 Scan 때 파일이 그대로 있어 **되살아납니다**. "시동만 한
> 로그"를 영구히 치우려면 이 **Delete flight**(파일 삭제)를 쓰세요.

### 5.5 Logged Messages
비행 중 기록된 시스템 메시지(Armed, Takeoff, Landing 등)를 시간·레벨과 함께 표.
(스크롤 시 헤더 고정)

## 6. Flights 탭 — 여러 개 한 번에 (일괄 지정)

같은 날 시동만 한 로그 여러 개에 **조종사·기체를 한 번에** 지정할 때.

1. 각 카드 **왼쪽 체크박스**를 선택 (또는 리스트 헤더 체크박스로 **현재 페이지 전체**).
   - 선택은 **페이지를 넘어가도 유지**됩니다.
2. 위에 뜨는 **일괄 바**에서 **Pilot 입력** / **Assign vehicle 선택**.
   - 입력한 항목만 적용됩니다(예: 조종사만 입력하면 조종사만).
3. **Apply** → 선택한 모든 로그에 반영. **Clear** 로 선택 해제.

---

## 7. Flights 탭 — CSV 내보내기

- 리스트 헤더의 **Export CSV** 버튼.
- **현재 필터(기체·스택·날짜·검색)가 적용된 모든 로그**가 CSV로 다운로드됩니다.
  (체크박스 선택과 무관 — 화면에 보이는 조건 전체)
  - 예: *스택=PX4 + 6월 날짜 + 검색=특정 조종사* → 그 조건의 기록만 내보내기.
- 파일은 **Excel에서 바로** 열리며(한글 안 깨짐), 날짜·위치·기체·조종사·비행시간·거리·
  속도·상태·파일명·비고 등 열을 포함합니다.

## 8. 자동으로 처리되는 것 (신경 안 써도 됨)

- **중복 방지**: 같은 로그가 **이름만 다르거나 다른 포맷**으로 들어와도 한 번만 등록됩니다.
  - ① 내용 해시(파일 바이트가 같은 경우) ② 같은 컨트롤러(hwid)+같은 시작시각.
  - 스캔 결과에 `dup N`으로 표시.
- **유령 항목 정리**: 스캔할 때 **디스크에서 사라진 파일**의 목록 항목을 자동 제거합니다
  (폴더 이동/이름변경으로 생긴 중복 방지). 결과에 `removed N missing`.
- **시작 위치(시군구)**: 시작 GPS 좌표를 역지오코딩해 시/군/구를 자동으로 채웁니다.

## 9. Aircrafts 탭 — 기체 관리

### 9.1 기체란?
로그가 들어오면 **컨트롤러(hwid)별로 기본 기체**가 자동 생성됩니다. 여기에 **등록번호·
별명·상태**를 붙여 관리합니다.

### 9.2 기체 추가 (수동)
- 상단의 **Registration # / Nickname** 입력 후 **+ Add aircraft**.
- 로그 없이도 미리 등록해 둘 수 있습니다(예: 새로 들여온 기체).

### 9.3 등록정보 수정
- 각 카드에서 **Registration # / Nickname / Notes** 수정 후 **Save**.
- 등록번호를 지정하면 Flights 탭의 기체 필터가 등록번호 기준으로 동작합니다.

### 9.4 상태 (Active / Maintenance / Retired)
- **Status** 드롭다운으로 지정. 제목 옆에 색 배지로 표시되고, **Retired** 기체는 카드가
  흐리게 보입니다.
  - 🟢 **Active**(운용중) · 🟡 **Maintenance**(정비중) · ⚪ **Retired**(은퇴)

### 9.5 기체 삭제
- 카드의 **Delete** 버튼. 그 기체에 **배정된 비행이 있어도 삭제 가능**합니다.
- 삭제하면 그 비행들은 **원래 컨트롤러의 기본 기체로 되돌아가거나**(또는 미할당) 처리되어,
  **비행 자체는 사라지지 않습니다.**

### 9.6 컨트롤러를 다른 기체로 옮겨 달았을 때 (스왑 워크플로) ⭐
같은 픽스호크를 A기체 → B기체로 옮겨 단 경우:

1. **Aircrafts**에서 **B기체를 추가**(또는 기존 기체 선택).
2. **Flights**로 가서, **스왑한 날짜 구간**으로 날짜 필터를 겁니다.
3. 헤더 체크박스로 **전체 선택** → 일괄 바에서 **Assign vehicle = B기체** → **Apply**.
4. 끝. 이후 **재스캔해도 그 배정은 유지**됩니다.

> 신규 비행은 일단 컨트롤러 기준 기본 기체로 들어오므로, 스왑 때마다 위처럼 **날짜로
> 묶어 한 번에** 올바른 기체로 옮기면 됩니다.

## 10. Pilots 탭 — 조종사 관리

- 상단 **Pilot name** 입력 → **+ Add pilot** 로 명단(로스터)에 추가.
- 각 조종사별 **출격 수 · 총 비행시간 · 기체 수**와 기체별 내역이 집계됩니다.
- **Delete**: 명단에서 제거. 단, **비행이 1건이라도 있으면 삭제 불가**(먼저 그 비행들의
  조종사를 바꾸거나 비워야 함) — 데이터 보호를 위해서입니다.

## 11. 자주 묻는 질문 (FAQ)

**Q. 같은 로그가 2개씩 보여요.**
보통 폴더를 옮기거나 이름을 바꿔 옛 경로의 항목이 남은 경우입니다. 다시 **Scan**하면
사라진 파일의 항목이 자동 정리되고, 내용이 같은 중복도 자동으로 합쳐집니다.

**Q. 시동만 하고 안 띄운 로그를 지우고 싶어요.**
상세에서 **Delete flight**(파일까지 삭제). 여러 개면 날짜 필터 후 일괄로 고른 뒤 하나씩
지우거나, 우선 한 기체로 모아두고 관리하세요.

**Q. 시간이 이상해요.**
시각은 **GPS 기준 UTC**로 저장되고 화면엔 로컬 시간으로 표시됩니다. CSV는 UTC로
나갑니다. GPS가 없는 로그(`partial`)는 절대 시각/트랙/거리가 없습니다.

**Q. 삭제·업로드가 안 돼요.**
NAS 공유폴더(`/logs`)에 컨테이너가 **쓰기/삭제 권한**이 있어야 합니다. 권한이 없으면
목록에선 지워져도 파일은 안 지워질 수 있습니다(관리자 문의).

## 12. 데이터 보관

- 모든 입력(조종사/특이사항/기체/상태)과 파싱 캐시는 로그 폴더의 **`flightlogbook.db`**
  한 파일에 저장됩니다. **로그 폴더를 통째로 옮기면 기록도 함께** 따라갑니다.
- 안전을 위해 입력이 있을 때마다 같은 위치에 **`flightlogbook.db.bak`** 백업이
  자동 생성됩니다.
